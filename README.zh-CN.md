# 逐字输入（Automatic Filling）

Chrome 扩展（Manifest V3）：在任意网页选择输入框后，将预设文本以可调速度逐字输入，支持标点停顿、光标始终在末尾，且不覆盖已有内容。

### 演示

![演示动图](demo.gif)

---

## 功能

- **选择输入框**：点击「开始选择并输入」后，在页面上点击目标输入框（含 `input`、`textarea`、`contenteditable`），即开始在该处追加输入。
- **不覆盖内容**：在已有内容末尾追加输入，不清空原内容。
- **光标在末尾**：输入过程中光标始终保持在内容末尾。
- **速度可调**：通过滑块设置「字/分钟」（约 30～300），线性控制输入间隔。
- **标点停顿**：勾选「标点后停顿」时，遇到标点符号会多停约 0.35 秒，更贴近自然节奏。
- **内容持久化**：插件内输入框的内容会保存，下次打开弹窗时自动恢复。

---

## 安装

1. 打开 Chrome，访问 `chrome://extensions`。
2. 打开右上角 **开发者模式**。
3. 点击 **加载已解压的扩展程序**，选择本项目目录（含 `manifest.json` 的 `automaticFilling` 文件夹）。

---

## 使用步骤

1. 打开要操作的网页。
2. 点击扩展图标，在弹窗中：
   - 在文本框内输入或粘贴要自动输入的内容（支持多行）；
   - 用滑块设置「字/分钟」；
   - 按需勾选「标点后停顿」。
3. 点击 **开始选择并输入**，弹窗会关闭。
4. 回到网页，点击目标输入框。
5. 扩展会在该输入框末尾逐字输入，可随时点击扩展图标再点 **停止** 中止。

> 不支持在 `chrome://`、`edge://`、应用商店等受限制页面使用，请先在普通网页上操作。

---

## 技术说明

- **选择器**：记录输入框时优先用 `#id`，其次用 `name`，否则用简短 CSS 路径，以便在同一页面稳定定位。
- **事件**：输入时触发 `input` / `change`，并尽量兼容 React 等受控组件（通过原生 value setter 写入）。
- **存储**：使用 `chrome.storage.local` 保存弹窗内文本与上次使用的配置。
